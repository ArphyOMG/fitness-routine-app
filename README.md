2. **문제 정의 및 목표**
    
    
    | **구분** | **내용** |
    | --- | --- |
    | **핵심 문제** | 운동 시 '뭘 해야 하지?'와 '자세가 맞나?' 때문에 루틴이 흐트러지고 기록이 끊긴다. |
    | **타겟 유저** | 헬스 초~중급자 (루틴은 있으나 기록/수행이 들쭉날쭉), 웹 우선 사용 |
    | **성공 기준** | 5명 테스트 시 3명 이상이 3일 내 2회 이상 기록 / '운동 선택 → 기록 저장' 30초 내 완료 |
3. **핵심 기능 명세**
    
    
    | **#** | **기능** | **상세 설명** | **우선순위** |
    | --- | --- | --- | --- |
    | 1 | **운동 목록 / 검색** | 16개 운동 시드 데이터 제공. 이름/부위 텍스트 검색, 카테고리 필터(전체/푸시/풀/하체/코어) | 🔴 P0 — 필수 |
    | 2 | **운동 상세 패널** | 카드 클릭 시 우측 슬라이드 패널. 이름/카테고리/주동근/장비/설명/자세팁 3개/주의사항 2개 표시 | 🔴 P0 — 필수 |
    | 3 | **운동 기록 입력** | 날짜(기본 오늘) + 세트별 횟수/무게 입력. 세트 추가/삭제. 저장 버튼. localStorage 영구 저장 | 🔴 P0 — 필수 |
    | 4 | **기록 이력 조회** | 운동별 최근 5개 이력 조회 (상세 탭). 전체 이력은 날짜별 그루핑으로 별도 페이지 제공. 개별 삭제 가능 | 🟡 P1 — 중요 |
    | 5 | **반응형 UI** | 모바일(768px 이하) 대응. 사이드 패널이 전체 화면으로 전환. 2열 그리드로 압축 | 🟡 P1 — 중요 |
4. **화면 구성 (3개 뷰)**
    
    
    | **화면** | **구성 요소** | **기능 설명** | **비고** |
    | --- | --- | --- | --- |
    | **(A) 홈/리스트** | 헤더, 검색창, 카테고리 필터, 운동 카드 그리드 | 운동 카드 클릭 시 상세 패널 오픈. 검색/필터 실시간 반영 | 메인 진입 화면 |
    | **(B) 상세 패널** | 3탭: 설명 / 기록하기 / 기록 이력 | 설명 탭: 운동 정보 + '기록하기' 버튼. 기록 탭: 세트 폼. 이력 탭: 최근 5개 | 우측 슬라이드 패널 (420px) |
    | **(C) 전체 이력** | 날짜별 그루핑, 운동명 + 세트 태그 | 헤더 '기록 이력' 버튼으로 진입. 날짜 내림차순 정렬. 각 로그 삭제 가능 | 전환 시 패널 닫힘 |
5. **데이터 구조**
    
    
    | **엔티티** | **필드** | **타입** | **설명** |

7. **LogStore 인터페이스 (Firebase 확장 대비)**
    
    현재 localStorage 구현. 동일 함수 시그니처로 Firebase로 교체 가능.
    
    | **함수** | **반환** | **설명** |
    | --- | --- | --- |
    | getLogs() | Log[] | 전체 로그 반환 |
    | getLogsByExercise(exerciseId) | Log[] | 운동별 로그 최신 5개 |
    | getAllLogsByDate() | [date, Log[]][] | 날짜별 그룹핑된 전체 로그 |
    | addLog(log) | void | 로그 추가 후 localStorage 저장 |
    | deleteLog(logId) | void | 해당 id 로그 삭제 후 저장 |
